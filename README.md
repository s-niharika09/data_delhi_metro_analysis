# Delhi Metro Data Analysis

## Project Overview
This project analyzes Delhi Metro trip data to understand passenger demand, revenue trends, route popularity, and travel conditions using **Python**, **Pandas**, and **SQL**.

## Objectives
- Analyze passenger demand across different stations and time periods.
- Identify revenue trends based on travel conditions.
- Determine the most popular routes.
- Examine ticket usage patterns.
- Clean and preprocess raw data for accurate analysis.

## Dataset
- Source: Delhi Metro trip data
- Format: CSV / SQL Database
- Features include:
  - `TripID` – Unique identifier for each trip
  - `Journey_Date` – Date of the trip
  - `From_Station` – Starting station
  - `To_Station` – Ending station
  - `Distance_km` – Distance of the trip in kilometers
  - `Passengers` – Number of passengers
  - `Ticket_Type` – Type of ticket used
  - `Fare` – Fare collected
  - `Remarks` – Travel condition (Peak, Offpeak, Festival, Weekend)

## Data Cleaning
- Standardized station names (trimmed spaces, uppercase).
- Handled missing values using modes/median where applicable.
- Removed duplicates and inconsistent entries.

## Analysis Performed
1. **Passenger Demand** – Total and average passengers by station, route, and month.
2. **Revenue Trends** – Revenue generated by different ticket types and travel conditions.
3. **Route Popularity** – Most traveled routes and peak/offpeak analysis.
4. **Ticket Usage** – Distribution of ticket types used by passengers.
5. **Travel Conditions** – Insights into peak, offpeak, weekend, and festival travel patterns.

## Tools & Libraries
- Python 3.x
- Pandas
- MySQL (for querying the dataset)
